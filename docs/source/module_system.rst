Module System Enhancement
===========================

pwncat-vl includes enhanced module system features.

Pattern Matching
----------------

Find modules by pattern using the ``match`` command:

.. code-block:: bash

    # Find all escalation modules
    match escalate.*
    
    # Find Linux enumeration modules
    match linux.enumerate.*
    
    # Find modules with platform filter
    match enumerate.* -p linux

Programmatic Access
-------------------

Use the Manager's ``match_modules`` method:

.. code-block:: python

    # Match modules by pattern
    for module in manager.match_modules("escalate.*"):
        print(module.name)
    
    # Match with platform filter
    for module in manager.match_modules("enumerate.*", platform=manager.target.platform):
        print(module.name)

Module Context
--------------

The module context system allows you to work with modules more efficiently.

Entering Module Context
~~~~~~~~~~~~~~~~~~~~~~~

Use the ``use`` command to enter a module context:

.. code-block:: bash

    use linux.enumerate.system.distro

This will:
- Set the module as the current context
- Show module information and arguments
- Allow running the module without specifying the name

Setting Module Arguments
~~~~~~~~~~~~~~~~~~~~~~~~~

In module context, use ``set`` to configure arguments:

.. code-block:: bash

    use linux.enumerate.system.distro
    set output_format json
    run

Running Modules
~~~~~~~~~~~~~~~

In module context, you can run the module without specifying the name:

.. code-block:: bash

    use linux.enumerate.system.distro
    run

Exiting Module Context
~~~~~~~~~~~~~~~~~~~~~~~

Use ``back`` to exit module context:

.. code-block:: bash

    back

Module Information
------------------

The ``use`` command now shows detailed module information:

- Module description
- Available arguments
- Argument types and defaults
- Help text for each argument

~~~~~~~~~~~~~~~~~~~~~~

Start a SOCKS5 proxy (default):

.. code-block:: bash

    socks -p 1080

Start a SOCKS4 proxy:

.. code-block:: bash

    socks -p 1080 -v 4

Managing Proxies
~~~~~~~~~~~~~~~~

List all active SOCKS proxies:

.. code-block:: bash

    socks -l

Stop a proxy:

.. code-block:: bash

    socks -s 1080

Usage Examples
--------------

Using SOCKS proxy with other tools:

.. code-block:: bash

    # Start SOCKS proxy in pwncat
    socks -p 1080
    
    # Use with curl
    curl --socks5-hostname 127.0.0.1:1080 http://internal-server.local
    
    # Use with proxychains
    proxychains nmap -sT -Pn 10.10.10.0/24
